## Variant A: Calm Document

### Design Stance
An editorial, document-focused view. The left side is a clean, document-like editor while the right side integrates the system diagram as a visual, high-contrast block layout.

### Key Choices
- **Layout:** Balanced split-pane with deep margins and comfortable spacing.
- **Color Palette:** Warm, stone-based gray hues (`bg-stone-50`, `bg-white`) evoking a clean notebook or documentation page.
- **Typography:** Premium sans-serif (`Inter`) for interface/prose, coupled with code-oriented mono (`JetBrains Mono`) for specifications.
- **Feel:** Editorial, professional, clean.

### Trade-offs
- **Strong at:** Readability of the spec, clean look, excellent for sharing or printing out.
- **Weak at:** Massive multi-layered systems, which may feel constrained by the document aesthetic.

### Best For
Product managers, engineering leads, or architects who want a high-fidelity visual documentation page that remains deeply human-readable.
